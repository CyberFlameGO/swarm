Introduction
Swarm is a program for the real-time analysis of seismic waveform data. Currently it can load data from Earthworm Wave Servers and the under-development Winston system.

This is a beta version of Swarm. There are missing features, known bugs, and other limitations to this version.
Please email me bug reports or feature requests. No bug report is too minor (such as a minor UI quirk). My email address is: dcervelli@stanford.edu

The current version is: 1.1.12.20050425
Important note #1: Channel names in Wave Servers (both WWS and Earthworm) are now space separated instead of underscore separated. This means that your SwarmGroups.config and SwarmKiosk.config files may need to change.

Important note #2: Version 1.1.12 fixes a bug that was causing Earthworm Wave Server data sources to not start in kiosk mode properly.

Important note #3: We have now finalized the communication protocol for Winston Wave Servers (WWS) and there have been changes. Consequently, version 1.1.10 will NO LONGER WORK with the pubavo1.wr.usgs.gov WWS. Please upgrade to 1.1.12 immediately.

Recent changes of note:
Swarm can access Winston Wave Servers.
Swarm can now read and write SAC files on the clipboard.

NOTE: There have been some minor changes to the Swarm.config file so it is recommended that you do not reuse your old Swarm.config file. I apologize for this and will make every effort to avoid changing formats in the future.

A list of bug fixes and changes can be found in Appendix C of the manual.

Installation
1. Install Java Runtime Environment (or SDK) v1.4+. Available at http://java.sun.com/j2se/1.4.2/index.jsp
Note: The recently released Java 1.5 or (Java 5.0) can also be used to run Swarm.

2. Download Swarm.
zip: http://www.stanford.edu/~dcerv/swarm/swarm-bin.zip
tar: http://www.stanford.edu/~dcerv/swarm/swarm-bin.tar.gz

3. Unzip or untar. The program files are located under a directory called 'swarm' so it is safe to unzip wherever you like.

4. Run Swarm. On Windows, simply run 'swarm.bat' in the 'swarm' directory. On Unix-like platforms, run the 'swarm.sh' script.

5. Go through the tutorial. This will quickly get you started using Swarm.
